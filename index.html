This guide will walk you through setting up Claude Code — Anthropic's AI coding assistant that lives right in your terminal. No prior coding experience is required. Before we dive in, let's make sure you have everything you need.

Module 1

Getting Started

~10 min

What you'll learn

  • Choose the right Claude plan and set up your account
  • Install Node.js and Claude Code on your machine
  • Navigate to a project folder and launch Claude Code from the terminal
  • Authenticate and start your first AI-assisted coding session

Step 1: Purchase a Claude Pro Plan

Claude Code is only available on paid plans — the free tier doesn't include it. Your subscription is what gives you access.

Go to claude.ai and sign up for a Pro, Max, Team, or Enterprise plan. This is the account you'll use to log in later.

Which plan should I choose?
For most individuals, the Pro plan is the best starting point. It gives you full access to Claude Code and is the most affordable paid option. The Max plan offers higher usage limits if you plan to use Claude Code heavily throughout the day.
Can I use Claude Code with a free account?
No. Claude Code requires a paid subscription (Pro, Max, Team, or Enterprise). The free tier of Claude only includes the web chat interface at claude.ai — it does not include Claude Code access.

Step 2: Download and Install Node.js

Node.js is a program your computer needs in order to run Claude Code. Think of it like installing a language your computer needs to speak.

Go to: https://nodejs.org/en/download

Pick the version for your operating system (Windows, Mac, or Linux), open the downloaded file, and follow the installer prompts. Make sure you install version 18 or newer.

How do I check if Node.js is already installed?
Open your terminal and type node --version. If you see a version number like v18.x.x or higher, you're good to go. If you get "command not found," you need to install it.
What if I have an older version of Node.js?
Claude Code requires Node.js 18 or newer. If your version is older, download the latest LTS version from nodejs.org and install it — it will replace the old version automatically.

Step 3: Open Your Terminal

A terminal is a text-based app where you type commands instead of clicking buttons. It looks like a blank window with a blinking cursor.

On Mac: Search for "Terminal" in Spotlight (press Cmd + Space, then type Terminal).

On Windows: Search for "Command Prompt" or "PowerShell" in the Start menu.

On Linux: Press Ctrl + Alt + T or search for Terminal in your app menu.

Is the terminal the same as the command line?
Yes — "terminal," "command line," "command prompt," and "shell" are often used interchangeably. They all refer to the text-based interface where you type commands. On Mac it's called Terminal, on Windows it's Command Prompt or PowerShell.

Step 4: Install Claude Code

In your terminal window, type the following command and press Enter:

npm install -g @anthropic-ai/claude-code

This downloads Claude Code onto your computer so it's ready to use from anywhere. The -g flag means it installs globally, not just for one project.

I'm getting a "permission denied" error. What do I do?
On Mac or Linux, try running the command with sudo in front: sudo npm install -g @anthropic-ai/claude-code. You'll be prompted for your computer password. On Windows, try opening your terminal as Administrator (right-click → "Run as administrator").
What does "npm" mean?
npm stands for Node Package Manager. It's a tool that comes bundled with Node.js and lets you install software packages (like Claude Code) from the internet with a single command.

Step 5: Navigate to Your Project Folder

Type cd followed by the path to your code folder. This tells Claude Code which files to work with.

cd Documents/my-project

If you're not sure of the path, you can drag your project folder into the terminal window on most systems and it'll paste the path automatically.

What if I don't have a project folder yet?
You can create one! In your terminal, type mkdir my-project to create a new folder, then cd my-project to enter it. Claude Code can work in an empty folder — you can ask it to generate files from scratch.

Step 6: Launch Claude Code

Type the following command and press Enter:

claude

This opens an interactive chat session right inside your terminal. You'll see a prompt where you can start typing instructions to Claude.

Can I run Claude Code from any folder?
Yes, but Claude Code works best when you launch it from inside the project you want to work on. It uses your current directory to understand which files are relevant. You can always switch projects by exiting (/exit) and navigating to a different folder.

Step 7: Sign In to Your Account

The first time you launch Claude Code, you'll be asked to log in with the same Anthropic account you used to purchase your plan in Step 1.

Follow the on-screen prompts to authenticate. This links your terminal session to your paid subscription.

Do I need to sign in every time I use Claude Code?
No. Once you sign in the first time, your session is saved and you won't be asked again unless your session expires or you explicitly log out.

Step 8: Start Talking to Claude

You're all set! Just type what you need in plain English. For example:

  • "Fix the bug in my login page"
  • "Write a function that sorts a list"
  • "Explain what this file does"

Claude will read, create, and edit your files directly from the terminal.

Can Claude Code actually modify my files?
Yes — Claude Code can read, create, edit, and delete files in your project. It always asks for permission before making changes, so you can review what it wants to do before accepting.
Is there a limit to how much I can use it?
Usage depends on your plan. The Pro plan includes a generous daily allowance. If you hit the limit, you can upgrade to Max for higher caps, or simply wait until it resets. You can check your usage anytime by typing /usage.

Why Use an IDE with Claude Code?

While Claude Code works great in a plain terminal, pairing it with a modern IDE gives you a much better experience. An IDE (Integrated Development Environment) is a code editor with built-in tools like file browsing, syntax highlighting, and a terminal — all in one window.

Google Antigravity is a free, AI-first IDE built on top of VS Code. It has native support for Claude models (including Claude Opus and Sonnet) and was designed from the ground up for AI-assisted coding. Here's why it's a great companion for Claude Code:

  • 💻 Built-in terminal — Run Claude Code right inside the IDE without switching windows
  • 👀 See your files and code side by side — Watch Claude's edits appear in real time as it works
  • 🤖 Agent mode — Antigravity can orchestrate multiple AI agents working in parallel on your project
  • 🆓 Free for individuals — Available on Mac, Windows, and Linux at no cost
  • 🔀 Model flexibility — Use Claude, Gemini, or other models depending on the task

If you're new to coding, an IDE like Antigravity makes everything less intimidating — you get a visual file tree, clickable errors, and a friendlier interface than a raw terminal.

Full documentation: https://docs.claude.com/en/docs/claude-code/overview

What you'll learn

  • Sign in and explore the Agent Manager and workspace
  • Configure AI models including Claude Opus and Sonnet

What is Antigravity?

Google Antigravity is a free code editor (IDE) designed for AI-assisted development. It's built on top of VS Code — so if you've used VS Code before, it'll feel familiar — but it adds powerful AI features on top, including native support for Claude models.

Think of it as a workspace where you can write code, browse your files, run your terminal, and talk to AI agents — all in one window.

Step 1: Go to the Antigravity Website

Visit antigravity.google in your browser.

You'll see download options for your operating system.

Is Antigravity the same as VS Code?
Antigravity is built on top of VS Code, so the interface looks and feels similar. However, it adds AI-native features like an Agent Manager, built-in model selection, and deeper integration with AI tools — things that regular VS Code doesn't have out of the box.

Step 2: Download for Your Operating System

Click the download button for your platform:

  • 🍎 Mac — Download the .dmg file
  • 🪟 Windows — Download the .exe installer
  • 🐧 Linux — Download the .deb or .rpm package

The download is free for individual use — no payment or credit card required.

Is Antigravity really free?
Yes — Antigravity is free for individual use. There's no payment or credit card required to download and use it. The built-in AI models (including Gemini and Claude) are also included at no cost.

Step 3: Install and Launch Antigravity

On Mac: Open the .dmg file and drag Antigravity into your Applications folder. Then open it from Applications or Spotlight (Cmd + Space, type "Antigravity").

On Windows: Run the .exe installer and follow the prompts. Antigravity will be added to your Start menu.

On Linux: Install the package using your package manager (e.g., sudo dpkg -i antigravity.deb on Ubuntu/Debian).

When you first open Antigravity, you'll see the welcome screen with options to switch to the Agent Manager, code with an Agent, or enter edit-only mode:

What's the difference between Agent Manager, Code with Agent, and Edit mode?
Agent Manager is your home base for managing AI conversations and workspaces across projects. Code with Agent opens the editor with an AI assistant ready to help. Edit mode is a plain code editor without AI — useful if you just want to write code yourself.

Step 4: Sign In and Explore the Agent Manager

When you first open Antigravity, you'll be prompted to sign in with a Google account. This gives you access to the built-in AI features and Gemini models at no cost.

After signing in, the Agent Manager lets you manage workspaces, start conversations with agents, and open the editor. It's your mission control for working with AI across multiple projects:

Do I need a Google account specifically?
Yes — Antigravity uses Google sign-in to authenticate you and provide access to the built-in AI features. Any Google account (including personal Gmail) will work.

Step 5: Choose Your AI Model

Antigravity supports multiple AI models. Use the model selector dropdown below the conversation input to switch between models:

  • 💎 Gemini 3 Pro — Google's most capable model (default)
  • ✨ Claude Sonnet 4.6 (Thinking) — Anthropic's Claude for thoughtful, detailed work
  • 🧠 Claude Opus 4.6 (Thinking) — Anthropic's most capable Claude model
  • 🌐 GPT-OSS-120B — OpenAI's open-source model

Simply click the model name in the dropdown to switch. No API key needed — all models are included free.

Which model should I pick?
It depends on the task. Gemini 3 Pro is the default and great for general use. Claude Opus 4.6 is the most capable for complex reasoning and large codebases. Claude Sonnet 4.6 is a good balance of speed and quality. You can switch models anytime mid-conversation.
Do I need an API key to use these models?
No. All models listed in the Antigravity model selector are included free with your Google sign-in — no separate API keys or billing setup required.

Step 6: Install the Claude Code Extension

Antigravity supports extensions through the Open VSX marketplace. To install Claude Code:

  1. Open the Extensions panel (Cmd + Shift + X on Mac, Ctrl + Shift + X on Windows/Linux)
  2. Search for "Claude Code"
  3. Click Install on the Claude Code extension by Anthropic

If the extension isn't available on Open VSX, you can install it manually:

  • Download the .vsix file from Anthropic's releases
  • In the Extensions panel, click the ... menu → Install from VSIX
  • Or run: antigravity --install-extension claude-code.vsix from your terminal
Why install the Claude Code extension if Antigravity already has AI built in?
Antigravity's built-in AI uses Google's Gemini models by default. The Claude Code extension gives you access to Anthropic's Claude as a dedicated coding agent with its own chat panel, terminal CLI mode, inline diffs, and plan review — features specifically designed for software engineering workflows.
What if the extension isn't available on Open VSX?
You can install it manually by downloading the .vsix file from Anthropic's GitHub releases. Then use the Extensions panel's "..." menu → "Install from VSIX," or run antigravity --install-extension claude-code.vsix in the terminal.

Step 7: Use Claude Code in Antigravity

Once installed, Claude Code integrates directly into Antigravity in two ways:

  • 💬 Chat panel — Open the Claude Code sidebar (click the Claude icon in the activity bar) to have conversations, ask questions about your code, and request edits. Claude can see your open files and workspace context.
  • ⌨️ Terminal CLI — Open the built-in terminal (Ctrl + ` or View → Terminal) and type claude to launch Claude Code in agent mode. As Claude reads and edits files, the changes appear in the editor in real time.

The advantage of running Claude Code inside Antigravity is that you get side-by-side editing — you can watch Claude's changes happen live in the editor while interacting with it in the terminal or chat panel, with no window switching needed.

Should I use the chat panel or the terminal CLI?
The chat panel is great for beginners — it provides a visual interface with inline diffs and plan review. The terminal CLI is more powerful for advanced users who want full control, tab completion, and all available commands. You can use both side by side.
Can Claude Code see all my files?
Claude Code can see files in your current workspace and any files you explicitly share via @-mentions. It always asks for permission before editing files. Your code is processed to provide assistance but is not used to train models.

Your Tools: IDE, Terminal & Desktop

~8 min

What you'll learn

In Modules 1 and 2, you installed Claude Code and set up Antigravity. Now let's take a closer look at the three places you can use Claude Code — the IDE, the standalone terminal, and the desktop app — so you know what each one does and when to reach for it.

The IDE at a Glance

When you open Antigravity with a project folder, you'll see a window divided into several areas. Don't worry about memorizing everything — think of this as a quick orientation so you know where to look.

Here are the five main areas you'll use:

  • 🗂️ Activity Bar (far left) — A thin strip of icons. Click an icon to open different panels, like the file explorer, search, extensions, or Claude's chat.
  • 📁 File Explorer (left panel) — Shows all the files and folders in your project as a tree. Click any file to open it.
  • 📝 Editor Pane (center) — The big area where you read and write code. Open files appear as tabs at the top.
  • ⌨️ Terminal Panel (bottom) — A built-in terminal where you can run commands — including claude — without leaving the IDE.
  • 💬 AI Chat Panel (right side) — Claude's conversation interface. Ask questions, request edits, and see suggestions here.
Do I need to memorize all these areas?
Not at all. You'll naturally learn where things are as you use the IDE. The two areas you'll use most are the Editor Pane (where your code lives) and the Terminal Panel (where you talk to Claude Code). Everything else is there when you need it.
This looks a lot like VS Code. Is it the same?
Almost. Antigravity is built on top of VS Code, so the layout and most features are identical. The main additions are the built-in AI chat panel, the Agent Manager, and native support for multiple AI models — things you'd need to install separately in regular VS Code.

The File Explorer & Editor

The File Explorer is the panel on the left that shows your project's files and folders as a tree. It works just like the file browser on your computer — click a folder to expand it, click a file to open it in the editor.

The Editor Pane is the large center area where your files open. A few things to notice:

  • Tabs — Each open file gets a tab at the top. Click tabs to switch between files, or close them with the small × button.
  • Syntax highlighting — The editor color-codes your files so different parts of the code are easy to distinguish. This isn't something you need to set up — it happens automatically.
  • Line numbers — Every line is numbered on the left side, making it easy to find specific spots when Claude says something like "look at line 42."

This is where you'll see Claude's work in real time. When Claude edits a file, the changes appear right here — you can watch the code update live.

What is syntax highlighting?
Syntax highlighting means the editor uses colors to make code easier to read. For example, keywords might appear in purple, text strings in green, and numbers in orange. You don't need to do anything to enable it — the editor detects the file type and applies colors automatically.
Can I have multiple files open at the same time?
Yes! Each file opens in its own tab. You can have as many tabs open as you need and click between them freely. You can even split the editor to see two files side by side — right-click a tab and choose "Split Right" or "Split Down."

The Terminal Panel

The terminal panel sits at the bottom of the IDE. If you don't see it, press Ctrl + ` (that's the backtick key, usually above Tab) or go to View → Terminal.

This is the same kind of terminal you used in Module 1 — it's just built right into the IDE so you don't need to switch windows. Here's how to read it:

  • The prompt — The line that ends with $ or %. This is where you type commands. It usually shows your current folder name.
  • Commands — What you type after the prompt. Press Enter to run a command.
  • Output — Everything that appears after you run a command. This is the computer's response.
  • Knowing when it's done — When you see a new prompt appear, the previous command has finished. If the terminal seems stuck (no new prompt), the command is still running.

To use Claude Code here, just type claude and press Enter — exactly like you did in Module 1. Claude's interactive session will start right inside this panel.

How do I know when a command is done running?
Look for a new prompt line (ending in $ or %) to appear. That means the terminal is ready for your next command. If you see text still scrolling or no new prompt, the command is still working. You can also look for a blinking cursor at the start of a new line.
Can I have multiple terminals open?
Yes! Click the + button in the terminal panel to open a new terminal tab. This is useful when you want Claude Code running in one terminal and want a second terminal for other commands — like running a web server or checking file status.

The Standalone Terminal

You don't always need the full IDE. The standalone terminal — the one you used back in Module 1 (Terminal on Mac, Command Prompt or PowerShell on Windows) — works just as well for running Claude Code.

Everything Claude can do in the IDE terminal, it can do in the standalone terminal too. The commands are identical: claude to start a session, type your instructions, and Claude reads and edits your project files.

The main difference? You won't see a file explorer or live code editing beside it. But for quick tasks — like asking Claude a question, running a one-off command, or working in a folder you haven't opened in the IDE — the standalone terminal is faster to reach for.

Is the standalone terminal different from the IDE terminal?
Functionally, no. They're the same thing — a terminal where you type commands. The IDE just embeds one inside the editor window for convenience. The standalone version is the app you open directly from your computer (Terminal on Mac, Command Prompt on Windows).
When should I use the standalone terminal instead of the IDE?
Use the standalone terminal when you want to do something quick without opening a full project — like asking Claude a general question, working in a folder you haven't set up as a project, or when you just prefer a simpler, distraction-free interface.

Claude Code Desktop

Claude Code Desktop is a standalone app you can download for Mac or Windows. It gives you a clean, visual chat interface for working with Claude — no terminal knowledge required.

Think of it as the friendliest way to use Claude Code. It looks and feels like a messaging app:

  • 💬 Chat interface — Type instructions in plain English, just like texting. Claude responds with explanations, code, and file changes.
  • 📂 Project aware — Point it at a folder and Claude can read, create, and edit files in that project — same as the terminal version.
  • 👀 Visual diffs — When Claude wants to change a file, you see a clear, color-coded diff showing exactly what's being added, removed, or changed before you approve it.
  • 🚀 No terminal needed — You never need to type commands like cd or npm. Just open the app, choose a folder, and start chatting.

You can download it from claude.ai/code — look for the Desktop App option. Sign in with the same Anthropic account you set up in Module 1.

Do I need a separate account for the Desktop app?
No. The Desktop app uses the same Anthropic account and Claude Pro subscription you already set up in Module 1. Sign in with the same credentials and you're ready to go.
Can the Desktop app do everything the terminal version can?
Almost everything. The Desktop app can read files, write code, create projects, and explain code — all through a visual interface. A few advanced features (like background agents and some slash commands) work best in the terminal version, but for day-to-day use, the Desktop app is fully capable.

When to Use What

You now have three ways to use Claude Code. Here's a quick guide to help you pick the right one:

The IDE (Antigravity)

Best for: Working on a project where you want to see your files, code, and Claude's edits all in one window. The most complete experience.

The Terminal

Best for: Quick tasks, power users, and when you want full control. Lightweight — no extra apps to open. Works everywhere.

Claude Code Desktop

Best for: Beginners, quick questions, and anyone who prefers a visual chat experience. No terminal knowledge needed.

There's no wrong choice here. Many people use all three depending on the situation — the IDE for focused project work, the terminal for quick one-off tasks, and the Desktop app when they want the simplest possible experience. Pick whichever feels most comfortable and you can always switch later.

Can I switch between tools in the middle of a project?
Absolutely. All three tools work with the same project files on your computer. You can start a task in the Desktop app, then open the same project in the IDE to see the file tree, or switch to the terminal for a quick command. Claude will see the same files regardless of which tool you use.

You've set up your tools and you know where everything lives. Now let's go deeper into how Claude Code works — the concepts that turn it from a simple chatbot into a powerful assistant that understands your project and takes action on your behalf.

What Is an Agent?

You've probably used AI chatbots before — you type a question, you get an answer. An agent is different. An agent doesn't just talk — it does things.

When you use Claude Code, Claude acts as an agent. That means it can:

  • 📖 Read your files — Claude can look at any file in your project to understand your code, data, or documentation.
  • ✏️ Write and edit code — Claude can create new files, modify existing ones, and fix bugs — all from a single instruction.
  • ⚙️ Run commands — Claude can execute terminal commands on your behalf, like installing packages, running tests, or starting a server.
  • 🧠 Make decisions — Claude can figure out the next step on its own. If fixing a bug requires reading three files, editing two, and running a test, Claude will do all of that without you spelling out each step.

The key difference: a chatbot gives you answers. An agent gives you results.

Is an agent the same as a chatbot?
No. A chatbot (like the free version of Claude at claude.ai) only responds with text. An agent can take real actions — reading files, writing code, running commands, and making multi-step decisions. Think of a chatbot as someone who gives you directions, and an agent as someone who drives you there.
Will Claude make changes without asking me first?
By default, Claude asks for your permission before editing files or running commands. You'll see exactly what it wants to do and can approve or reject each action. You're always in control.

Agent Mode & Plan Mode

Claude Code has two ways of working — think of them as two gears:

Agent Mode (the default) is the "just do it" gear. When you give Claude an instruction, it immediately starts working — reading files, making edits, running commands — and asks for your permission at each step. This is what you've been using so far.

Plan Mode is the "think first" gear. Instead of jumping into action, Claude creates a detailed plan — a step-by-step outline of what it intends to do — and shows it to you before making any changes. You can review the plan, suggest adjustments, and then give the go-ahead.

To switch to Plan Mode, type:

/plan

Or simply tell Claude: "Plan this out for me before doing anything." Claude will show you its proposed approach, and you can say "looks good, go ahead" or ask for changes.

When should I use Plan Mode vs. Agent Mode?
Use Agent Mode (default) for straightforward tasks where you're confident in the outcome — "fix this typo," "add a button here," "explain this file." Use Plan Mode for bigger or more complex tasks where you want to review the approach first — "refactor the authentication system," "set up a new feature," or anything where the wrong approach could create a mess.
What if I don't like Claude's plan?
Just tell Claude what to change! You can say things like "skip step 3," "use a different approach for the database," or "simplify this — I don't need all those steps." Claude will revise the plan based on your feedback. You're the final decision-maker.
Can I switch between modes?
Yes. You can switch to Plan Mode anytime by typing /plan, and back to Agent Mode by giving a direct instruction. You don't need to restart your session — just change how you'd like Claude to work on the current task.

Background Agents

Sometimes you want Claude to work on a task while you do something else — like sending it off to do research while you grab coffee. That's what background agents are for.

A background agent runs in the cloud — not on your computer. You give it a task, and it works on it independently. You can close your terminal, shut your laptop, and come back later to check the results.

To start a background agent from the terminal:

claude --background "your task description here"

For example: claude --background "review this project and list all the TODO comments"

You can also start background agents from the Desktop app or the web at claude.ai/code.

Do background agents cost extra?
Background agents use the same Claude Pro (or Max) subscription you already have. They count toward your normal usage limits — no extra charge. Max plan users get higher limits, which is helpful if you plan to run background agents frequently.
How do I check on a background agent?
You can check the status of your background agents at claude.ai/code in the "Agents" section, or by running claude --resume in your terminal to see a list of recent sessions. When a background agent finishes, you'll see its results — including any files it created or modified.

Skills & Slash Commands

Slash commands are quick shortcuts that start with /. Instead of typing out a full instruction, you type a short command to trigger a specific action. Think of them as keyboard shortcuts, but for conversations.

Here are the most useful slash commands to start with:

  • /help — See all available commands and what they do
  • /plan — Switch to Plan Mode (think before acting)
  • /compact — Summarize the conversation to free up memory when it gets long
  • /clear — Start a fresh conversation (clears the current chat history)
  • /usage — Check how much of your daily usage you've consumed
  • /model — Switch between AI models (e.g., Sonnet for speed, Opus for depth)
  • /review — Ask Claude to review recent code changes

You don't need to memorize these — just remember that /help exists, and it will show you everything else.

Do I need to memorize all these commands?
No! The only one you really need to remember is /help — it lists all available commands with descriptions. Over time, you'll naturally remember the ones you use most often. You can also just talk to Claude in plain English for almost anything these commands do.
Can I create my own slash commands?
Yes! You can create custom slash commands (called skills) by adding instruction files to your project's .claude/skills/ folder. Each file becomes a new command you can invoke by name. This is an advanced feature — start with the built-in commands first.

Context: CLAUDE.md & Rules

Imagine starting a new job with zero context — no onboarding document, no team wiki, no one explaining the project. You'd figure things out eventually, but it would be slow. CLAUDE.md is like an onboarding document for Claude.

CLAUDE.md is a file you create in your project's root folder. Claude reads it automatically every time you start a session. It tells Claude things like:

  • 📋 Project conventions — "We use TypeScript, not JavaScript" or "All functions should have JSDoc comments"
  • 🏗️ Architecture decisions — "The API lives in /src/api/" or "We use PostgreSQL for the database"
  • ⚠️ Things to avoid — "Never modify files in /legacy/" or "Don't use jQuery"
  • 🔧 How to build and test — "Run npm test to run the test suite" or "Use make build to compile"

To create one, just make a file called CLAUDE.md in your project folder and write your instructions in plain English. You can even ask Claude to help you write it: "Create a CLAUDE.md for this project."

For more granular control, you can also create a .claude/rules/ folder with separate files for different topics — like security.md for security rules or testing.md for testing conventions. Claude reads these automatically too.

Where do I put CLAUDE.md?
Put it in the root (top level) of your project folder — the same folder you cd into before running claude. Claude looks for it there automatically. For example, if your project is in Documents/my-project/, create Documents/my-project/CLAUDE.md.
What should I write in CLAUDE.md?
Start simple. Write 3-5 sentences about your project: what it does, what language/framework it uses, and any important rules. For example: "This is a React app using TypeScript. Run tests with npm test. Don't modify anything in the /generated/ folder." You can always add more later as you discover what's useful.
Do I need CLAUDE.md to use Claude Code?
No — Claude Code works perfectly fine without it. But adding a CLAUDE.md makes Claude significantly more effective because it doesn't have to guess at your conventions. Think of it as optional but highly recommended, especially for larger projects.

Putting It All Together

Here's how all these concepts come together in a real workflow:

  1. Pick your tool — Open the IDE for project work, the terminal for quick tasks, or the Desktop app for a visual experience.
  2. Claude reads your context — When the session starts, Claude automatically reads your CLAUDE.md and any rules files, so it already understands your project's conventions.
  3. Give your instruction — Describe what you need in plain English. Claude acts as an agent — reading, writing, and running commands to get it done.
  4. Steer with slash commands — Use /plan to review an approach, /compact when the conversation gets long, or /model to switch to a different AI model mid-task.
  5. Send long tasks to the background — For tasks that take a while, use a background agent so you can keep working on other things.

That's the complete picture. You have the tools, the modes, the commands, and the context system. The best way to learn from here is to start using it — pick a project, open your favorite tool, and try asking Claude to do something real.
